# ТЕХНОЛОГИЧЕСКИЙ СТЕК ПРОЕКТА
Сервис для создания индивидуального рациона работника предприятия с помощью ИИ

### 1. Backend и API-сервер
• **FastAPI** — основной фреймворк для создания REST API. Принимает данные о сотруднике (тип задач, физические параметры, ограничения по здоровью, предпочтения), взаимодействует с базой данных и моделью ИИ, формирует и отдаёт персональные рекомендации. Выбран за высокую скорость, асинхронность и автоматическую генерацию документации.
• **Uvicorn** — высокопроизводительный ASGI-сервер. Запускает и обслуживает FastAPI в асинхронном режиме, обеспечивает стабильную работу при тестовых нагрузках (10–20 одновременных пользователей).
• **Pydantic** — библиотека валидации данных. Гарантирует корректность входящих JSON-запросов (вес, рост, аллергии, уровень нагрузки), предотвращает ошибки на уровне API и упрощает преобразование типов.
• **SQLModel/ SQLAlchemy** — ORM-слой для работы с базой данных. Позволяет описывать таблицы (блюда, сотрудники, рекомендации, фидбек) через Python-классы, выполнять CRUD-операции без написания сырых SQL-запросов и безопасно передавать параметры.

### 2. База данных
• **SQLite** — легковесная файловая СУБД. Идеальна для MVP: не требует установки отдельного сервера, хранит меню столовой, профили сотрудников, историю выбранных блюд и логи обратной связи. Легко переносится между средами разработки и тестирования.
• **DB Browser for SQLite** — визуальный редактор базы данных. Используется на этапах проектирования и отладки для просмотра структуры таблиц, проверки связей, ручного добавления тестовых блюд и контроля целостности данных.

### 3. Машинное обучение и работа с данными
• **scikit-learn** — основная библиотека машинного обучения. Используется для обучения базовой модели рекомендаций (например, решающее дерево, случайный лес или гибридный скоринг на основе правил+ оптимизация). Подбирает рацион с учётом суточной нормы ккал/БЖУ, типа профессиональной нагрузки и медицинских ограничений.
• **pandas& numpy** — инструменты обработки и математических расчётов. Нормализуют параметры сотрудников, рассчитывают пищевую ценность блюд, формируют обучающие выборки, агрегируют данные фокус-группы для анализа метрик качества рекомендаций.
• **Faker** — генератор синтетических данных. Создаёт реалистичные профили сотрудников, параметры нагрузок, аллергии и предпочтения для обучения и тестирования модели без использования реальных персональных данных.
• **joblib** — инструмент сериализации. Сохраняет обученную модель ИИ в компактный файл .pkl, позволяя FastAPI быстро загружать её при старте и отдавать прогнозы без повторного обучения.

### 4. Мобильное приложение (клиентская часть)
• **Kivy & Kivy MD** — фреймворк для кроссплатформенной разработки графического интерфейса на Python. Kivy MD добавляет компоненты Material Design. Реализует ключевые экраны: регистрация/профиль, выбор типа задач и нагрузки, отображение дневного рациона, отметка приёма пищи и отправка фидбека.
• **Buildozer** — инструмент автоматической сборки. Компилирует Python-код с Kivy в готовый .apk файл для Android. Позволяет развернуть прототип на реальных смартфонах сотрудников для проведения фокус-группы.
• **httpx/ requests** — HTTP-клиент внутри мобильного приложения. Отправляет запросы к FastAPI (получение рекомендаций, передача предпочтений, отправка оценок), поддерживает асинхронную работу для сохранения плавности интерфейса.

### 5. Инструменты разработки, тестирования и документации
• **Git& GitHub** — система контроля версий и хостинг кода. Обеспечивает совместную работу команды, ветвление (main, dev, feature), фиксацию изменений, тегирование релиза v1.0-MVP и безопасное хранение кода.
• **Swagger UI** — автоматическая интерактивная документация. Генерируется FastAPI по адресу /docs и /redoc. Служит единым контрактом данных для синхронизации frontend/backend и используется при демонстрации проекта.
