# Устав проекта

## 1) название проекта
Сервис для создания индивидуального рациона работника предприятия с помощью ИИ

## 2) Краткое описание проекта
Многие рабочие в наши дни не следят за своим питанием на работе, что приводит к серьезным проблемам со здоровьем.

Наш сервис должен создавать индивидуальные рационы на основе меню предприятия, выполняемых задач и индивидуальных особенностей сотрудников.

## 3) цель проекта
Разработать мобильное приложение, которое создает индивидуальные рационы на основе меню предприятия, выполняемых задач и индивидуальных особенностей сотрудников.

## 4) задачи проекта
- Провести анализ существующих систем организации меню и рационов сотрудников предприятий.
- Разработать модель ИИ для подбора индивидуального рациона сотрудника на основе его физических нагрузок, состояния здоровья и личных предпочтений
- Создать базу данных блюд с их пищевой ценностью и ингредиентами
- Разработать прототип мобильного приложения
- Интегрировать систему с базой данных столовой
- Провести тестирование прототипа на фокус группе сотрудников

## 5) актуальность проекта
Питание напрямую влияет на работоспособность и здоровье сотрудников, однако на большинстве предприятий оно организовано по шаблону, без учета индивидуальных особенностей и профессиональных нагрузок. Это приводит к росту заболеваемости, снижению производительности и увеличению числа ошибок.

Технологии Индустрии 4.0 — искусственный интеллект и анализ больших данных — позволяют персонализировать рацион на основе реальных потребностей каждого работника. Внедрение такого сервиса дает предприятию инструмент для повышения эффективности труда и профилактики профессиональных рисков. При этом готовых решений, адаптированных под промышленные предприятия и их инфраструктуру питания, на рынке практически нет.

## 6) целевая аудитория
Работники крупных предприятий, заводов, фабрик

## 7) технологии при разработке
При разработке проекта планируется использовать технологии искусственного интеллекта, технологии больших данных.

## 9) план создания проекта с контрольными точками

### 1. Аналитика и проектирование
Сбор требований к сервису, анализ существующего меню столовой, проектирование структуры базы данных, разработка алгоритмов работы ИИ.

**Срок:** 19.03.2026 – 26.03.2026

**Результат:** утвержденный документ с требованиями, схема базы данных, описание выбранных алгоритмов ИИ и принципов персонализации рациона.

### 2. Разработка MVP (общий срок: 26.03.2026 – 16.04.2026)

#### a. Разработка базового API на Python
Создание REST API для приёма данных о сотруднике (задачи, здоровье, предпочтения) и выдачи рекомендаций по рациону.

**Срок:** 26.03.2026 – 16.04.2026

**Результат:** работоспособность всех запланированных эндпоинтов, корректный обмен данными между API и будущим мобильным приложением.

#### b. Обучение простой модели ИИ на синтетических данных
Генерация синтетических данных (сотрудники, блюда, пищевая ценность, нагрузки), обучение базовой модели для подбора индивидуального рациона.

**Срок:** 26.03.2026 – 16.04.2026

**Результат:** модель выдает осмысленные рекомендации на тестовых наборах данных, качество подбора (метрики точности/ релевантности) не ниже заданного порога.

#### c. Создание прототипа с 1-2 ключевыми функциями
Разработка минимального интерфейса мобильного приложения (выбор блюд из меню, отображение персонального рациона).

**Срок:** 26.03.2026 – 16.04.2026

**Результат:** приложение успешно получает рекомендации от ИИ, пользователь может просмотреть предложенный рацион и отметить выбор.

### 3. Интеграция и тестирование
Подключение прототипа к базе данных столовой (блюда, состав, калории), проведение функционального и нагрузочного тестирования, исправление критических ошибок.

**Срок:** 16.04.2026 – 23.04.2026

**Результат:** прототип корректно загружает актуальное меню из БД столовой, нет критических сбоев при одновременной работе 10–20 тестовых пользователей.

### 4. Презентация прототипа
Подготовка демонстрационных материалов (презентация, сценарий показа), тестирование на фокус-группе из 5–7 сотрудников, сбор обратной связи.

**Срок:** 23.04.2026 – 30.04.2026

**Результат:** успешная публичная демонстрация работы приложения, заполненные анкеты с отзывами фокус-группы, итоговый отчёт о готовности прототипа к доработке.
